# Conversion Quality Guide

## General Principles

> **This project focuses on AI-friendly document conversion, not pixel-perfect layout reconstruction.**

> **本项目重点是 AI 友好的文档转换，而不是像素级版式还原。**

## Format-Specific Notes

### Markdown → PDF / DOCX / HTML

- **Quality**: Excellent. Pandoc produces high-quality output for these conversions.
- **PDF engine**: PDF export quality depends on the selected PDF engine. Different engines produce different results for complex layouts, fonts, and CJK text.
- **Tables**: Standard Markdown tables convert reliably.
- **Code blocks**: Syntax highlighting depends on Pandoc's built-in themes.
- **TOC**: Supported via `--toc` flag for PDF and DOCX.

### DOCX → Markdown

- **Quality**: Good for standard documents.
- **Images**: Embedded images may be extracted to a separate directory (`--extract-media`).
- **Complex layouts**: Multi-column layouts, floating images, and custom Word styles may not translate perfectly.
- **Recommendation**: Use `cleanForLLM: true` to get a cleaner output for AI consumption.

### PDF → Markdown

> **PDF → Markdown is content extraction, not layout or semantic structure reconstruction.**
>
> **PDF 转 Markdown 是内容提取，不是版式或语义结构还原。**

- **PDF files usually do not preserve Markdown semantics.**
- **普通 PDF 通常不保存 Markdown 语义。**
- **Tables, headings, code blocks, lists, and reading order may not be reliably recovered.**
- **标题、表格、代码块、列表、阅读顺序都可能无法可靠恢复。**

- **Text-based PDFs**: Variable results with MarkItDown or Pandoc. Basic text extraction works but structure is often lost.
- **Scanned PDFs**: Will NOT work without OCR. This server does not include OCR.
- **Complex layouts**: Two-column papers, complex tables, footnotes, and mathematical formulas may not convert reliably.
- **Sidecar recovery**: If the PDF was generated by this server with `preserveSource=true`, the original Markdown is available as a sidecar file (`.source.md`). Use `preferSourceSidecar: true` (default) to automatically recover it.

- **Recommendation**: Always use `engine: "markitdown"` for best text extraction. Set `cleanForLLM: true`. For accurate recovery, rely on sidecar files generated with `preserveSource=true`.

### Batch Conversion

- **Quality**: Same as individual conversions.
- **Tolerance**: Individual file failures do NOT abort the batch. Check the `results` array for per-file status.

## Engine Comparison

| Operation | Pandoc | MarkItDown |
|-----------|--------|------------|
| Markdown → PDF | Excellent | N/A |
| Markdown → DOCX | Excellent | N/A |
| Markdown → HTML | Excellent | N/A |
| DOCX → Markdown | Good | Good |
| PDF → Markdown | Fair | Better |

## Recommended Workflow for Accurate Recovery

1. When converting Markdown → PDF, set `preserveSource: true`.
2. This generates two sidecar files alongside the PDF:
   - `sample.pdf.source.md` — the original Markdown content
   - `sample.pdf.meta.json` — metadata about the conversion
3. When converting PDF → Markdown, `preferSourceSidecar: true` (default) will automatically find and return the original Markdown if the sidecar exists.
