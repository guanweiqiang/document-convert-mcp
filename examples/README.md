# Examples

This directory contains minimal examples for `@guan-tools/document-converter-mcp`.

## 1. Create a workspace

```bash
mkdir E:/MCPWorkDir
```

Copy `sample.md` into the workspace:

```bash
copy examples\sample.md E:\MCPWorkDir\sample.md
```

On macOS/Linux, use your own workspace path and update `DOC_CONVERTER_WORKSPACE`.

## 2. Start with MCP Inspector

Use `mcp.json` as a starting point.

```bash
npx @modelcontextprotocol/inspector --config examples/mcp.json --server document-converter
```

If the package name is still a placeholder, replace `@your-scope/document-converter-mcp` with the real npm package name.

## 3. Try tool requests

The `requests/` directory contains example inputs for each tool:

* `markdown_to_html.json`
* `markdown_to_docx.json`
* `markdown_to_pdf.json`
* `docx_to_markdown.json`
* `pdf_to_markdown.json`
* `batch_convert.json`

Copy the JSON content into MCP Inspector when calling the corresponding tool.

## Notes

### PDF export

For Chinese Markdown to PDF, use:

```json
{
  "pdfEngine": "xelatex",
  "cjkMainFont": "Microsoft YaHei"
}
```

You need Pandoc and a PDF engine such as MiKTeX or TeX Live.

### PDF to Markdown

PDF to Markdown is content extraction, not layout or semantic reconstruction.

It may not reliably preserve:

* headings
* tables
* code blocks
* lists
* reading order

For PDFs generated by this server, use:

```json
{
  "preserveSource": true
}
```

when calling `markdown_to_pdf`.

Then `pdf_to_markdown` can use the generated `.source.md` sidecar file for accurate recovery.

### Do not commit generated outputs

Generated files such as `.pdf`, `.docx`, `.html`, and converted `.md` files should stay in your workspace and should not be committed as examples.
